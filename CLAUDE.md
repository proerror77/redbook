# Redbook 内容生产系统

## 项目概述
这是一个系统化的内容生产工作流，用于管理多平台（小红书、抖音、X.com、公众号）的内容创作。

## 目录结构
```
01-内容生产/          # 内容生产流水线
  ├── 选题管理/       # 选题记录和管理
  ├── 01-待深化的选题/ # 已确定要做的选题
  ├── 02-制作中的选题/ # 正在制作的内容
  └── 03-已发布的选题/ # 已发布归档

02-内容素材库/        # 可复用素材
  ├── 核心概念库/     # 理论框架
  ├── 金句库/         # 高质量表达
  ├── 爆款文稿库/     # 已验证的内容
  └── 案例库/         # 案例素材

03-方法论沉淀/        # 数据驱动的方法论
04-内容数据统计/      # 发布数据记录
05-选题研究/          # 选题分析
06-业务运营/          # 商业数据
```

## 🎯 标准化内容生产工作流

### 阶段 1️⃣：选题研究

#### 「研究 X.com 话题」
当用户说「研究 X」「X 研究」或想要从 X.com 获取选题灵感时：
- **使用 Skill**：`/x-collect`
  - 自动搜索话题、分析热门趋势、提取痛点
  - 生成结构化研究报告
  - 推荐高价值选题
- **自动化方式**（已配置每日定时任务）：
  ```bash
  python3 tools/auto-x/scripts/daily_research.py
  ```
  - 每日 7:00 AM 自动运行
  - 输出：`05-选题研究/X-每日研究-{日期}.md`

#### 「发现创业机会」
当用户说「Reddit 痛点挖掘」「发现创业机会」时：
1. **选择目标社区**：r/SaaS, r/Entrepreneur, r/startups, r/webdev
2. **运行分析脚本**：
   ```bash
   python3 tools/reddit_hack.py <reddit_url> [output_file]
   ```
3. **评估机会**：市场规模、支付意愿、竞争情况、实现难度
4. **转化为选题**：发现有价值的痛点后，使用「记录选题」

#### 「记录选题」
当用户提供一个想法或说「记录选题」时：
1. 添加到 `01-内容生产/选题管理/00-选题记录.md` 的「待处理选题」
2. 格式：`- [ ] 选题想法 | 来源/灵感 | 日期`

---

### 阶段 2️⃣：内容创作

#### 「深化选题」
当用户说「深化选题 [选题名]」时：
1. **检索素材库**（关键步骤）
   - 搜索 `02-内容素材库/核心概念库/` 查找理论框架
   - 搜索 `02-内容素材库/金句库/` 查找高质量表达
   - 搜索 `01-内容生产/03-已发布的选题/` 查找相关文稿
2. **建议复用**：如果找到相关内容，先询问是否复用
3. **生成文稿**：基于素材库生成新文稿
4. **保存**：保存到 `01-内容生产/01-待深化的选题/`

#### 「检索素材」
当用户说「检索素材 [关键词]」时：
1. 搜索 `02-内容素材库/` 下所有子目录
2. 列出相关的概念、金句、案例
3. 建议如何在当前内容中使用

#### 「生成标题」
当用户说「生成标题」或「生成小红书标题」时：
1. 读取 `03-方法论沉淀/标题方法论.md` 获取标题公式
2. 生成 3 个标题选项，说明逻辑和预期效果
3. 提供封面文字建议

#### 「优化开头」
当用户说「优化开头」时：
1. 读取 `03-方法论沉淀/短视频开头方法论.md`
2. 分析当前开头的问题
3. 提供 2-3 个优化版本，说明钩子类型

#### 「创作 X 推文」
当用户说「写推文」「创作 X 内容」时：
- **使用 Skill**：`/x-create`
  - 基于选题生成病毒式推文
  - 支持单条推文、Thread、长文
  - 自动优化文案和排版

---

### 阶段 3️⃣：制作发布

#### 「生成小红书图文」
当用户说「生成小红书图文」「制作小红书卡片」时：
- **唯一方式**：`/baoyu-xhs-images` skill
  - 支持 10 种视觉风格 x 8 种布局
  - 自动分析内容、生成大纲、渲染图片
  - **包含发布功能**（无需额外脚本）

#### 「发布到 X.com」
当用户说「发布到 X」「发布推文」时：
- **唯一方式**：`/baoyu-post-to-x` skill
  - 支持文本、图片、视频、长文章
  - 通过 Chrome CDP 创建草稿
  - **注意**：目前 `--submit` 参数有 bug，不会自动点击发布按钮
- **正确流程**：
  1. 使用 skill 创建草稿（会自动打开预览）
  2. 在预览页面手动点击右上角「发布」按钮
  3. 避免重复运行脚本（会创建重复草稿）
- **发布后提醒记录数据**

#### 「一稿多发」
当内容创作完成后，使用**发布清单**确保多平台发布：

**发布清单模板**：
```markdown
## 发布清单 - {选题名}

- [ ] X.com（长文/Thread）
- [ ] 小红书（图文）
- [ ] 抖音（短视频，可选）
- [ ] 公众号（长文，可选）
```

**标准流程**：
1. 创作完成后，生成发布清单
2. 逐个平台发布，勾选完成
3. 全部发布后，统一「记录数据」
4. 「发布归档」

---

### 阶段 4️⃣：数据复盘

#### 「记录数据」
当用户提供发布数据时：
1. 添加到 `04-内容数据统计/数据统计表.md`
2. 如果是爆款，询问是否添加到爆款文稿库
3. 提醒更新方法论（如有新发现）

#### 「发布归档」
当用户说「发布归档 [文稿名]」时：
1. 从 `02-制作中的选题/` 移动到 `03-已发布的选题/`
2. 在文稿末尾添加发布信息（日期、平台）
3. 提醒记录数据

---

## 📦 工具说明

### 主要 Skills（推荐使用）
- **X.com 研究**：`/x-collect` - 话题搜索、趋势分析、痛点提取
- **X.com 创作**：`/x-create` - 病毒式推文生成
- **X.com 发布**：`/baoyu-post-to-x` - 自动发布推文

- **筛选话题**：`/x-filter` - 选题评分和过滤（按需使用）

### 自动化脚本（后台任务）
**X.com 每日研究**（已配置 launchd 定时任务）：
- 位置：`tools/auto-x/scripts/daily_research.py`
- 运行时间：每日 7:00 AM
- 输出：`05-选题研究/X-每日研究-{日期}.md`
- 手动运行：`python3 tools/auto-x/scripts/daily_research.py`

**Reddit 痛点挖掘**：
- 位置：`tools/reddit_hack.py`
- 用法：`python3 tools/reddit_hack.py <reddit_url> [output_file]`
- 输出：痛点分析报告（Markdown）
