# 强制规则：时间意图自动捕获

> **类型**：强制规则（Procedural Rule）
> **来源**：从优化方法 10 中提炼
> **优先级**：P0（自动执行）
> **创建时间**：2026-02-15
> **最后更新**：2026-02-15

---

## 规则定义

当用户的话语中包含**时间意图**时，AI **必须自动捕获**并添加到任务清单，不得遗漏。

---

## 触发词识别（强制）

### 时间相关触发词

| 触发词 | 时间范围 | 优先级 | 示例 |
|--------|----------|--------|------|
| 明天 | 次日 | P0 | "明天发布这篇文章" |
| 今天 | 当日 | P0 | "今天完成这个任务" |
| 下周 | 下周某天 | P1 | "下周写一篇关于 AI 的文章" |
| 本周 | 本周内 | P1 | "本周完成 3 篇内容" |
| 周X | 具体星期 | P1 | "周五发布视频" |
| X号 | 具体日期 | P0 | "15号之前完成" |
| 月底 | 本月最后一天 | P1 | "月底复盘数据" |
| 稍后 | 今日内 | P2 | "稍后优化标题" |
| 等会 | 1-2 小时内 | P2 | "等会回复评论" |
| 记得 | 不明确（询问） | P1 | "记得发布" |

### 动作相关触发词

| 触发词 | 任务类型 | 示例 |
|--------|----------|------|
| 发布、上线 | 发布任务 | "明天发布" |
| 完成、做完 | 完成任务 | "下周完成" |
| 写、创作 | 创作任务 | "今天写文章" |
| 优化、改进 | 优化任务 | "稍后优化" |
| 回复、回应 | 互动任务 | "等会回复" |
| 分析、研究 | 分析任务 | "本周分析数据" |
| 学习、练习 | 学习任务 | "下周学习视频制作" |

## 执行流程（强制）

### 步骤 1：识别时间意图

**正则匹配**（示例）：
```regex
(明天|今天|下周|本周|周[一二三四五六日]|稍后|等会|记得)\s*(发布|完成|写|优化|回复|分析|学习)
```

**自然语言理解**：
- 提取动作：发布、完成、写、优化等
- 提取时间：明天、下周、15号等
- 提取对象：文章、视频、评论等

### 步骤 2：推断优先级

**自动判断**：
```
时间紧迫度 + 动作重要性 → 优先级

明天 + 发布 → P0（重要且紧急）
下周 + 学习 → P1（重要不紧急）
稍后 + 优化 → P2（紧急不重要）
```

### 步骤 3：添加到任务清单

**格式**：
```markdown
- [ ] {任务内容} | P{0-3} | {截止时间}
```

**文件位置**：`任务清单.md`

### 步骤 4：回复用户

**必须回复**：
```
✅ 已添加到任务清单
📅 截止时间：{时间}
⭐ 优先级：P{0-3}
📋 查看：任务清单.md
```

## 边界情况处理

### 情况 1：时间不明确
**触发条件**：用户说"稍后"、"等会"、"有空"

**处理方式**：
1. 询问具体时间：
   ```
   好的，大概什么时候？
   1. 今天
   2. 明天
   3. 本周内
   ```
2. 用户回复后，添加任务

### 情况 2：动作不明确
**触发条件**：用户说"明天处理这个"

**处理方式**：
1. 询问具体动作：
   ```
   好的，明天要：
   1. 发布
   2. 优化
   3. 完成
   4. 其他
   ```
2. 用户回复后，添加任务

### 情况 3：已存在相同任务
**触发条件**：任务清单中已有相同任务

**处理方式**：
1. 检查是否重复
2. 如果重复，询问：
   ```
   任务清单中已有"{任务}"，是否：
   1. 更新截止时间
   2. 添加新任务
   3. 忽略
   ```

## 自动完成任务

### 触发条件
用户说以下内容时，自动更新任务状态：

| 用户说法 | 任务状态 | AI 行动 |
|----------|----------|----------|
| "已完成 X" | 完成 | `[x]` + 记录完成时间 |
| "X 做完了" | 完成 | `[x]` + 记录完成时间 |
| "X 已发布" | 完成 | `[x]` + 提醒记录数据 |
| "取消 X" | 取消 | `[-]` + 标注原因 |
| "X 延期" | 延期 | `[>]` + 询问新时间 |

### 执行流程
1. 找到对应任务
2. 更新状态
3. 记录时间戳
4. 回复用户

**正确的意图捕获**：
- ✅ 识别了时间意图
- ✅ 添加到任务清单

- ❌ 遗漏时间意图

## 使用说明

**对于 AI**：
- **强制执行**：每次对话都必须扫描时间意图
- **不可遗漏**：即使时间不明确，也要询问确认
- **持续学习**：记录用户表达习惯，优化识别准确率

**对于用户**：
- 随口说出时间意图即可（例如："明天发布"）
- AI 会自动捕获并添加到任务清单
- 如果 AI 遗漏，直接提醒"你忘了添加任务"
