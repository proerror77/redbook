# 以前的 AI 入口

以前我们用 AI，大多是：

- 打开 ChatGPT
- 复制问题
- 等回答
- 再把答案搬回原来的工具

这像“问顾问”。

---

# 新的入口

现在 Agent 开始进入：

- 邮箱
- 浏览器
- IDE
- 本地代码图谱
- 语音界面
- 可穿戴设备

这像“进工位”。

---

# 邮箱场景

AI 不只是总结邮件。

它可以理解客户来信、找历史上下文、起草回复、标记风险。

但关键是：哪些邮件能看？哪些回复需要人确认？

---

# 浏览器场景

浏览器 Agent 可以读页面、填表、整理信息、操作后台系统。

但企业最该关心的是：

它能点哪些按钮？

能不能回滚？

日志在哪里？

---

# 代码场景

本地代码图谱让 Agent 不只是看单个文件。

它能理解模块关系、依赖、测试、提交历史。

这会改变工程协作，而不只是提高写代码速度。

---

# 语音和眼镜场景

语音让用户直接控制 App 状态。

眼镜和可穿戴设备让 Agent 变成随身工作界面。

入口越自然，权限边界越重要。

---

# 结论

聊天框解决回答。

工作流解决交付。

企业导入 AI，不是让每个人学更多 prompt，而是重新设计入口、权限、审批和回滚。
