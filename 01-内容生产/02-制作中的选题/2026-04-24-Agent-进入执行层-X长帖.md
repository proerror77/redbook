## 终稿

Agent 现在卷的已经不是“更聪明”了。

而是能不能在更长流程里自己 review、自己记忆、自己跑完。

这两天几个更新放一起看，这个信号非常明显：

- OpenAI 给 Codex 推了 Auto-review
- Anthropic 给 Claude Managed Agents 上了 memory
- 腾讯开源了 Cube Sandbox

表面上看，这是三家公司各发各的功能。

但底层其实是一回事。

大家都发现了：

agent 真正难的不是回答一次问题。

而是进到一个更长的流程以后，还能不能继续往前推。

比如：

- 它能不能自己 review
- 能不能记住上一次做了什么
- 能不能在云端继续跑
- 能不能有权限边界
- 能不能进浏览器、进沙箱、进工作流
- 出错以后能不能恢复

以前很多 agent 其实更像“高级聊天框”。

你问一次，它答一次。

答得好不好，是核心指标。

现在这个阶段开始变了。

大家在卷的是：

这个 agent 能不能真的像一个执行角色一样工作。

不是会不会说。

而是会不会做。

不是给你建议。

而是把流程往下推进。

不是跑一轮。

而是能不能在更长时间、更复杂边界里持续跑。

所以我觉得接下来 AI agent 的分水岭不会只是模型能力。

而是执行层能力：

- memory
- runtime
- sandbox
- browser
- review
- recovery
- observability

谁把这些补齐，谁的 agent 才更像产品。

补不齐，就还是 demo。
