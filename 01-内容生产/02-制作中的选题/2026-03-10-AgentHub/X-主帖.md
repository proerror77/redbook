大家现在聊 agent，大多还在卷单兵能力。

但 Karpathy 刚开的 AgentHub，我觉得更值得看的是另一层：

**如果以后不是 1 个 agent 干活，而是 10 个 agent 一起试、一起提交、互相接力，GitHub 这套 workflow 还顺手吗？**

这个 repo 的答案很克制：

- bare git repo
- message board
- 没有 main
- 没有 PR
- 没有 merge
- 只有 commit DAG

项目还很早。

但它提醒了一个更大的问题：

我们今天很多协作工具，默认服务的是“人类开发者”。

如果明天协作的基本单位变成一群 agent，
那工作流本身就要重写。

项目还很早，但方向值得看：
https://github.com/karpathy/agenthub
