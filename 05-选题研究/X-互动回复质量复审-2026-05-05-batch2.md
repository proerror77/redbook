# X 互动回复质量复审 - 2026-05-05 batch2

> 复审原因：用户反馈“很多回复不是很好，非常 AI”。  
> 结论：上一轮只做了脚本硬门槛审查，不等于真正通过 `x-mastery-mentor` 内容质量审稿。

## 复审标准

- 是否像真人在原帖下自然回复，而不是像运营模板。
- 是否贴原帖语境，而不是把无关内容硬转成“系统 / 工作流 / 产品 / 规则”。
- 是否对账号主线有帮助：AI agent、workflow、工具、ROI、工程协作、crypto 风控。
- 是否能引发对方继续回复，而不是只有抽象判断。
- 是否避免“这类 / 真正 / 最难 / 最值得 / 系统设计 / 底层结构”这类批量感表达。

## 总体判断

- 可保留：12 条
- 需要删除或重发：12 条
- 勉强可留但不建议作为后续风格样本：6 条

核心问题：
1. 过度使用“产品 / 规则 / 系统 / 工作流”视角，导致很多非 AI 帖被硬拧成咨询式评论。
2. 多条回复没有对原帖里的具体细节做回应，只做了抽象归纳。
3. 对娱乐、生活、社会新闻类帖子，账号主线不强，回复显得像为了完成 KPI。
4. “已审核”的口径不准确：它只是通过机械安全/格式 gate，不是高质量 X 审稿。

## 后续规则修正

1. 回复前不只做机械 review，还要做内容质量 review：`像不像真人`、`是否贴原帖细节`、`是否服务账号主线`。
2. 候选不足时不要硬凑 30 条；宁可只回 12 条高质量，也不要把娱乐/生活帖批量套成“系统观察”。
3. 对非账号主线帖，除非有非常自然的个人反应，否则不回。
4. 每批发布前抽样给用户看 5-10 条风格，确认后再批量发。
