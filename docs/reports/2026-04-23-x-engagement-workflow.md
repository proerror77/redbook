# X 互动工作流 - 2026-04-23

## 目标

账号仍处在静默期时，不应该只靠主帖等自然流量。

更有效的动作是每天去高质量相关帖下面留下判断，让账号进入别人已经形成的讨论场。

## 原则

### 1. 半自动，不全自动

默认只生成互动队列和评论草稿。

发布评论前仍需人工确认。

原因：

- 评论更容易显得像机器人
- 热点语境变化快
- 错评大号比发错自己主帖更伤账号

### 2. 只评论账号主线相关内容

优先主题：

- AI agent
- coding agent
- AI workflow
- 模型产品化
- Cursor / Claude Code / Codex / Kimi / Gemini / Cloudflare Agent Cloud
- 独立开发者和 AI 工具商业化
- 预测市场 / oracle / crypto infrastructure 只在有强观点时进入

跳过：

- 纯政治争吵
- 娱乐八卦
- 低质量 crypto shill
- 纯工具推荐合集
- 评论区已经爆炸且没有新增角度的大帖

### 3. 评论要像真人

默认结构：

```text
一句态度。

一句反直觉判断。

一句为什么。
```

可用结构：

```text
这点其实比新闻本身更关键。

真正变化不是 X，而是 Y。

因为 Z。
```

不要写：

- Great point
- 学到了
- 很有启发
- 收藏了
- 完全同意

### 4. 回复语言必须跟随原帖语言

- 英文帖：只用英文回复
- 中文帖：才用中文回复

原因：

- 评论首先要进入原评论区的自然语境
- 语言不一致会显得突兀，也会降低互动质量
- 账号默认语言不能压过原帖语言

## 每日节奏

建议每天 2-3 轮。

每轮：

- 扫 8-12 个候选帖
- 选 3-5 个高价值帖子
- 每个生成 2-3 条评论草稿
- 人工确认后发 1-3 条

日总量：

- 冷启动：5-8 条评论
- 稳定后：8-15 条评论
- 不建议超过 20 条

## 评分标准

总分 100。

- 主题匹配：35
- 帖子热度：20
- 互动窗口：15
- 可评论空间：20
- 风险扣分：最高 -30

优先评论：

- 大号/高质量创作者
- 刚起量但评论区还没满
- 能接上我们的独特判断
- 不需要查太多事实也能发表观点

## 推荐命令

```bash
python3 tools/auto-x/scripts/build_engagement_queue.py --limit 8
```

输出：

- `05-选题研究/X-互动队列-YYYY-MM-DD.md`
- `05-选题研究/X-互动队列-YYYY-MM-DD.json`

## 发布门槛

评论发布前检查：

- 是否像真人会直接说的话
- 是否有具体判断
- 是否没有模板味
- 是否不会引战到账号主线之外
- 是否值得挂在这个帖下面

## 后续升级

第一阶段：只生成草稿。

第二阶段：支持把用户选中的草稿复制到剪贴板或打开对应帖子。

第三阶段：只对低风险评论支持自动发，且每天限量。
