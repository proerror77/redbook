# 🚀 Redbook 快速入门

> 5 分钟掌握核心工作流

---

## 📝 核心概念

Redbook 是一个**多平台内容生产系统**，帮助你：
- 📊 自动研究热门话题（X.com、Reddit）
- ✍️ 智能创作内容（AI 辅助）
- 🚀 一键发布（小红书、X.com）
- 📈 数据驱动优化

---

## 🎯 三步开始

### Step 1: 选题研究

**方式 A：自动化（推荐）**
- 每天 7:00 AM 自动生成研究报告
- 位置：`05-选题研究/X-每日日程-{日期}.md`
- 查看后说「记录选题」保存感兴趣的话题

**方式 B：手动深挖**
```
你：研究 X 话题「AI 创业」
Claude：调用 /x-collect 分析话题
```

**方式 C：Reddit 痛点**
```bash
python3 tools/reddit_hack.py "https://reddit.com/r/SaaS/..."
```

### Step 2: 内容创作

**深化选题**：
```
你：深化选题「AI 时代的创业机会」
Claude：检索素材库 → 生成文稿草稿
```

**创作推文**：
```
你：创作 X 内容
Claude：调用 /x-create 生成病毒式推文
```

**制作小红书**：
```
你：生成小红书图文
Claude：调用 /baoyu-xhs-images 自动生成 + 发布
```

### Step 3: 发布复盘

**发布到 X**：
```
你：发布到 X
Claude：调用 /baoyu-post-to-x 全自动发布
```

**记录数据**：
```
你：记录数据（提供点赞、转发数）
Claude：添加到数据统计表 → 更新方法论
```

## ⚠️ 常见错误

### ❌ 不要做的事

1. **不要手动运行已集成的脚本**
   - ❌ `search_x.py`, `trending_topics.py`
   - ✅ 用 `bash tools/daily.sh` 或 `/x-collect`

2. **不要用弃用的工具**
   - ❌ `auto-redbook/render_simple.sh`
   - ✅ 用 `/baoyu-xhs-images`

3. **不要跳过素材库检索**
   - ❌ 直接生成新内容
   - ✅ 先「检索素材」，复用已有内容

### ✅ 正确做法

1. **研究阶段**：依赖自动化 + 手动补充
2. **创作阶段**：素材库先行，复用优先
3. **发布阶段**：全自动 Skills，避免手动
4. **复盘阶段**：每次都记录数据
