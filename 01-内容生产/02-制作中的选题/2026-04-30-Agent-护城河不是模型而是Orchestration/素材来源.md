## 核心来源

### 1. Cursor SDK

- Cursor Forum: `Cursor SDK in Public Beta`
- 时间：2026-04-29
- 核心点：
  - Cursor 把 agent 作为 TypeScript SDK 开放
  - 同一套 agent 可本地运行，也可跑在 cloud runtime
  - SDK 带着 codebase indexing、semantic search、MCP、skills、hooks、subagents 等运行时能力

### 2. Warp Open Source

- Warp Blog: `Warp is now open-source`
- 时间：2026-04-28
- 核心点：
  - Warp client 源码开放
  - License 为 AGPL
  - 开发流程强调由 Oz 支持的 agent-first 开发方式

### 3. Warp Universal Agent Support

- Warp Blog: `Universal Agent Support: Level Up Your Coding Agent in Warp`
- 时间：2026-04-14
- 核心点：
  - Warp 明确支持 Claude Code、Codex、Gemini CLI、OpenCode
  - 重点不是自有单一 agent，而是让多个 agent 在同一工作台中工作

### 4. Warp Oz / Agent Platform

- 核心点：
  - API / SDK
  - local agent + cloud agent
  - orchestration
  - observability
  - triggers
  - task lifecycle
  - environment and outputs

## 本文采用的判断边界

1. 不把 `Cursor SDK public beta` 写成 “Cursor 全量开源”。
2. 不把 `Warp open-source` 写成 “Warp 的全部云平台和 agent orchestration 都已开源”。
3. 本文不是测评谁更强，而是比较两种战略：
   - Cursor：开放 agent 出口
   - Warp：开放 agent 工作台 / 控制平面
4. 不做无法验证的市场份额、收入、融资推断。

## 文章主论点

本文要证明的不是 “模型不重要”。

而是：

当 agent 从一次回答走向长流程交付后，真正形成差异的层，会越来越从模型能力上移到 orchestration。

更具体地说，是这几个能力的组合：

- trigger
- context
- permission
- review
- handoff
- rollback
- observability
