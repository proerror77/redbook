# TELOS 模块化本体画像

> TELOS 是一个模块化的身份系统，将个人画像拆分为 7 个独立文件。
> 相比于单一的"个人定位.md"，TELOS 系统可以按需加载，大幅降低 token 消耗。

---

## 什么是 TELOS？

TELOS 代表 7 个核心模块：

```
T - Target (目标与规划)
E - Experience (背景与经历)
L - Life Principles (价值观与原则)
O - Operations (工作方式)
S - Skills (技能与能力)
+ Core Identity (核心身份)
+ Social Network (关系网络)
```

---

## 为什么需要模块化？

### 对比：第一代 vs 第二代

**第一代：个人定位.md（单一文件）**
- ❌ 每次都加载全部内容（~5000 tokens）
- ❌ 更新一项需要重读整个文件
- ❌ 难以维护和扩展

**第二代：TELOS 模块化（7 个文件）**
- ✅ 按需加载（只读需要的模块）
- ✅ 更新某项只需读取单个文件
- ✅ Token 消耗降低 70%+

---

## 模块说明

### 00-核心身份.md（Core Identity）⭐⭐⭐
**加载频率**：每次对话必读

**内容**：
- 一句话定位
- 核心特质（3-5 个关键词）
- 内容定位和目标受众
- 创作原则（要做的事/不做的事）

**Token 消耗**：~800 tokens

**使用场景**：
- 对话开始时
- 需要确认定位时
- 做重大决策时

---

### 01-价值观与原则.md（Life Principles）⭐⭐
**加载频率**：涉及决策时读取

**内容**：
- 核心价值观（真实性、用户价值、系统化思维等）
- 内容创作原则
- 商业原则
- 决策边界（可以做/不能做/需要谨慎）

**Token 消耗**：~1000 tokens

**使用场景**：
- 做决策时
- 遇到冲突时
- 接受商业合作时
- 需要判断边界时

---

### 02-背景与经历.md（Experience）⭐
**加载频率**：需要了解历史时读取

**内容**：
- 职业背景
- 内容创作经历
- 失败经验（重要！）
- 技能树
- 关键转折点

**Token 消耗**：~1200 tokens

**使用场景**：
- 需要了解过往经验时
- 给建议需要考虑背景时
- 规划发展路径时

---

### 03-技能与能力.md（Skills）⭐⭐
**加载频率**：分配任务时读取

**内容**：
- 核心能力矩阵（写作、设计、视频、AI、数据、技术）
- 详细技能清单
- 能力短板与提升计划
- 能力边界（能做/正在提升/不擅长）

**Token 消耗**：~1200 tokens

**使用场景**：
- 分配任务时
- 评估能力边界时
- 制定学习计划时
- 避免超出能力的任务

---

### 05-目标与规划.md（Target）⭐⭐
**加载频率**：制定计划时读取

**内容**：
- 3-5 年愿景
- 年度目标（OKR）
- 季度目标
- 月度目标
- 里程碑规划
- 能力提升计划

**Token 消耗**：~1500 tokens

**使用场景**：
- 制定计划时
- 评估任务优先级时
- 做长期决策时
- 定期复盘时

---

### 07-工作方式.md（Operations）⭐⭐⭐
**加载频率**：需要了解工作偏好时读取

**内容**：
- 工作节奏（每日/每周）
- 决策偏好
- 沟通偏好
- 工作流程偏好
- 自动化偏好
- 与 AI 的协作模式

**Token 消耗**：~1000 tokens

**使用场景**：
- 需要配合工作流程时
- 设计自动化方案时
- 理解沟通偏好时
- 明确协作边界时

---

### 08-关系网络.md（Social Network）⭐
**加载频率**：涉及人际关系时读取

**内容**：
- 核心协作者
- 同行创作者
- 用户/粉丝画像
- 商业合作伙伴
- 导师/顾问
- 社群与组织

**Token 消耗**：~1000 tokens

**使用场景**：
- 规划商业合作时
- 分析用户需求时
- 需要联系协作者时
- 涉及隐私保护时

---

## 加载策略

### 必读文件（每次对话）
- ✅ 00-核心身份.md

### 按需读取（根据任务）
- 涉及决策 → 01-价值观与原则.md
- 涉及任务分配 → 03-技能与能力.md
- 涉及工作流程 → 07-工作方式.md
- 涉及计划 → 05-目标与规划.md
- 涉及人际关系 → 08-关系网络.md
- 需要了解历史 → 02-背景与经历.md

## Token 消耗对比

### 第一代（单一文件）
```
个人定位.md: ~5000 tokens
每次对话都加载：5000 tokens
```

### 第二代（TELOS 模块化）
```
简单对话：800 tokens（只读核心身份）
普通任务：1800 tokens（核心身份 + 工作方式）
复杂决策：2800 tokens（核心身份 + 价值观 + 关系网络）
```

**节省率**：70%+ （大部分场景）

## 维护指南

### 更新频率建议

| 文件 | 更新频率 | 触发条件 |
|------|----------|----------|
| 00-核心身份.md | 每季度 | 定位发生变化 |
| 01-价值观与原则.md | 每季度 | 价值观调整、发现新边界 |
| 02-背景与经历.md | 每月 | 重要里程碑、失败案例 |
| 03-技能与能力.md | 每月 | 能力提升、发现短板 |
| 05-目标与规划.md | 每月 | 目标调整、完成里程碑 |
| 07-工作方式.md | 每季度 | 工作流程优化 |
| 08-关系网络.md | 每月 | 建立新关系、合作变化 |

### 更新流程

1. **记录触发事件**：什么事情导致需要更新
2. **确定更新模块**：需要更新哪个文件
3. **更新内容**：修改具体内容
4. **记录更新日志**：在文件末尾记录更新
