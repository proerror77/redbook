## 账号定位

- 主题：AI/大模型/论文解读/评测/硬件与算力
- 用户期待：快速得知"发生了什么"，一句话判断"值不值得看"

## 高互动特征

- 强钩子：标题像新闻标题，开头就给"事件/结论/疑问"
- 标点用力：`!`、`?` 高频，制造节奏与立场
- 数字化承诺：`18 张图告诉你怎么用`、`4 个新模型`
- 口语化+专业词混用：`突发`、`刚刚` + `SOTA`、`token`、`bench`
- 以"测试/对比/排名/结论"构建可信度

## 可套用句式

| 类型 | 模板 |
|------|------|
| 事件型 | `突发，{人/团队} {动作}。{一句补充}` |
| 解读型 | `{项目} 又发新东西了！给大家带来解读。` |
| 教程型 | `{产品} 发布！{N} 张图告诉你怎么用！` |
| 反问型 | `{问题} 吗？` |

## 结构配方

1. 一句话钩子（事件/问题/结论）
2. 一句话价值承诺（解读/实测/教程/总结）
3. 1-3 个要点（能用数字就用数字）
4. 结尾一句"继续讨论/提问/下集预告"

## 写作对齐检查单

- [ ] 第一行有没有钩子（事件/结论/反问）？
- [ ] 有没有给读者明确承诺（解读/实测/教程/总结）？
- [ ] 有没有数字化（N 点/N 张图/N 个模型）？
- [ ] 有没有"硬指标"（测试/对比/排名/数据）？
- [ ] 口语化是否足够，但不水？
