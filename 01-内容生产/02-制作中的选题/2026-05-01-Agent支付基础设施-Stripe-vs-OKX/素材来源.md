# 素材来源：Agent 支付基础设施对比

> 日期：2026-05-01

## 核心命题

这个选题真正要证明的是：Agent 支付不是钱包新闻，而是 AI Agent 进入真实商业流程前必须补齐的权限、凭证、结算和审计层。

它服务账号主线的方式是：把 Stripe 和 OKX 两个信号放在一起，说明企业导入 Agent 的关键问题正在从“模型会不会做”转向“谁授权、花多少钱、凭证怎么隔离、记录在哪里、出错怎么处理”。

## 已核验来源

- Stripe 官方：Giving agents the ability to pay
  - https://stripe.com/blog/giving-agents-the-ability-to-pay
  - 关键信息：Link agent wallet 允许用户授权 Agent 使用 Link；Agent 发起购买时生成一次性支付凭证；用户付款信息不会暴露给 Agent 或商户；现阶段每次付款请求需要用户 review，未来计划支持 spending limits。
- Stripe Sessions 2026 官方新闻稿
  - https://stripe.com/nl/newsroom/news/sessions-2026
  - 关键信息：Stripe 将 agentic commerce 放进一组 AI / stablecoin / billing 更新里，说明它不是孤立钱包功能。
- TechCrunch 报道
  - https://techcrunch.com/2026/04/30/stripe-link-digital-wallet-ai-agents-shopping/
  - 关键信息：补充外部媒体视角，确认 Link wallet 面向 AI agents shopping 场景。
- OKX 官方：OKX Launches Agent Payments Protocol
  - https://www.okx.com/learn/agent-payments-protocol-app
  - 关键信息：APP 面向 agent-to-agent commerce，包含 OKX Agentic Wallet、Payment SDK、X Layer 支付结算、按量计费、批量支付；roadmap 包含 escrow、dispute resolution、agent-to-human arbitration 等。

## 事实边界

- Stripe 更像法币/卡网络世界里的 Agent 授权支付层，当前重点是“人授权 Agent 支付”。
- OKX 更像链上/crypto-native 世界里的 Agent 商业协议，当前重点是“Agent 与 Agent / service 之间如何报价、支付、结算”。
- 两者都还处在基础设施早期阶段，不能写成“Agent 经济已经成熟”。
- OKX 的 escrow、争议处理等能力按官方描述属于 roadmap / future capability，文中不能当作已完整落地能力。

## 可用判断

- Agent 真正进入商业，不是因为它能聊天，而是因为它能在授权边界内执行有经济后果的动作。
- 支付只是表层，底层问题是权限模型。
- Stripe 解决的是“不要把银行卡交给 Agent”。
- OKX 解决的是“Agent 之间能不能形成可结算的商业关系”。
- 企业不会接受一个能随便花钱的 Agent，但会接受一个有额度、有审批、有日志、有撤销机制的 Agent。
