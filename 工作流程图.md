## 🔄 工具更新策略

### Skills 优先原则

当同一功能有多个工具选择时，优先使用 Skills：
1. **Skills 更新快**：由专业团队维护
2. **体验更好**：交互式、实时反馈
3. **稳定性高**：无需管理依赖

### 脚本定位

脚本仅用于：
1. **定时任务**：`bash tools/daily.sh`
2. **特殊场景**：`reddit_hack.py`
3. **调试工具**：保留但不主推

### 何时更新工具？

当出现以下情况时，考虑弃用旧工具：
- ✅ 新 Skill 功能更强
- ✅ 旧工具依赖复杂
- ✅ 使用频率低于 1 次/月
- ✅ 维护成本高

## 💡 最佳实践

### 1. 研究阶段

**定时 + 手动结合**：
- 依赖 `bash tools/daily.sh` 获取每日灵感
- 遇到感兴趣的话题，用 `/x-collect` 深挖
- Reddit 用 `reddit_hack.py` 定期扫描热门帖

### 2. 创作阶段

**素材库先行**：
- 先「检索素材」，复用已有内容
- 再「深化选题」，生成新文稿
- 最后用 Skills 润色和优化

### 3. 发布阶段

**全自动为主**：
- 优先使用 `/baoyu-*` Skills
- 避免手动操作，减少出错

### 4. 复盘阶段

**数据驱动**：
- 每次发布后立即「记录数据」
- 爆款内容添加到素材库
- 定期更新方法论

## 🆘 常见问题

### Q1: 为什么不用 auto-redbook 了？

**A**: `/baoyu-xhs-images` Skill 功能更强、更稳定、更易用。本地脚本依赖复杂（需要 agent-browser），而且发布流程需要手动操作。

### Q2: 每日自动化（tools/daily.sh）和 /x-collect 有什么区别？

**A**:
- `tools/daily.sh`：定时任务入口，自动化批量研究（内含提醒/回顾/研究）
- `/x-collect`：交互式，深度研究单个话题

### Q3: 我可以自定义 daily_research 的关键词吗？

**A**: 可以！编辑 `tools/auto-x/scripts/daily_research.py`，修改 `DEFAULT_KEYWORDS` 列表。

### Q4: Reddit 工具能分析中文社区吗？

**A**: `reddit_hack.py` 设计用于英文 Reddit。中文社区需要单独开发。

### Q5: 如何备份我的素材库？

**A**:
```bash
# 整个项目都在 Git 版本控制下
git add .
git commit -m "备份素材库"
git push
```
