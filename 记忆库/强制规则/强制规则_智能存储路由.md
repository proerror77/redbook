# 强制规则：智能存储路由

> **类型**：强制规则（Procedural Rule）
> **来源**：从优化方法 13 和实践经验中提炼
> **优先级**：P0（宪法级别）
> **创建时间**：2026-02-15
> **最后更新**：2026-02-15

---

## 规则定义

当用户说"保存 XXX"时，AI **必须自动判断存储位置**，不得询问用户"保存到哪里"。

---

## 执行流程（强制）

### 步骤 1：分析内容类型

根据关键词自动识别内容类型：

| 关键词 | 内容类型 | 存储位置 |
|--------|----------|----------|
| 文章、文稿、长文 | 文章内容 | `01-内容生产/02-制作中的选题/` |
| 选题、想法、灵感 | 选题想法 | `01-内容生产/选题管理/` |
| 素材、案例、概念 | 素材资源 | `02-内容素材库/` |
| 方法论、最佳实践 | 方法论 | `记忆库/语义记忆/` |
| 规则、原则、标准 | 强制规则 | `记忆库/强制规则/` |
| 日记、记录、总结 | 情景记忆 | `记忆库/情景记忆/` |

### 步骤 2：创建目录结构

```bash
# 如果是文章，自动创建目录
mkdir -p "01-内容生产/02-制作中的选题/YYYY-MM-DD-{标题}/"
```

### 步骤 3：保存文件

```bash
# 保存到确定的位置
echo "内容" > "{路径}/{文件名}.md"
```

### 步骤 4：告知用户

**必须包含的信息**：
1. ✅ 文件已保存
2. 📁 具体路径
3. 💡 决策理由（为什么保存到这里）

**回复模板**：
```
✅ 文章已保存到：01-内容生产/02-制作中的选题/2026-02-15-{标题}/
📁 内容类型：文章
💡 决策理由：长文本内容，用于发布
```

## 存储路由决策树

```
用户说"保存 XXX"
    ↓
分析关键词
    ↓
是否包含"文章/文稿/长文"？
    ├─ 是 → 01-内容生产/02-制作中的选题/
    └─ 否 ↓
是否包含"选题/想法/灵感"？
    ├─ 是 → 01-内容生产/选题管理/
    └─ 否 ↓
是否包含"素材/案例/概念"？
    ├─ 是 → 02-内容素材库/
    └─ 否 ↓
是否包含"方法论/最佳实践"？
    ├─ 是 → 记忆库/语义记忆/
    └─ 否 ↓
是否包含"规则/原则"？
    ├─ 是 → 记忆库/强制规则/
    └─ 否 ↓
默认 → 询问用户（仅当无法判断时）
```

## 禁止行为（绝对）

### ❌ 绝对禁止
1. **将文件保存到 redbook 目录外**（无任何例外）
2. **将文件保存到 `~/Dev/` 路径**
3. **在根目录直接保存文件**
4. **不告知用户保存位置**

### ⚠️ 需要确认
1. 删除已保存的文件
2. 移动已发布的内容
3. 覆盖现有文件

## 边界情况处理

### 情况 1：无法判断内容类型
**触发条件**：用户说"保存这个"，但没有明确内容类型

**处理方式**：
1. 分析上下文（最近在讨论什么）
2. 如果仍无法判断，询问用户：
   ```
   我需要确认一下内容类型，这是：
   1. 文章文稿
   2. 选题想法
   3. 素材资源
   4. 其他
   ```

### 情况 2：多个可能的存储位置
**触发条件**：内容同时符合多个类型

**处理方式**：
1. 选择最具体的类型
2. 例如："AI 工具使用案例" → `02-内容素材库/案例库/`（而非`02-内容素材库/`）

### 情况 3：用户明确指定路径
**触发条件**：用户说"保存到 XXX 目录"

**处理方式**：
1. 验证路径是否在 redbook 目录内
2. 如果是，使用用户指定路径
3. 如果不是，警告并使用默认路径

## 自我优化机制

### 每周复盘（周日）
1. 检查本周的存储决策
2. 识别错误存储案例
3. 更新路由规则

### 更新触发条件
- 发现错误存储 ≥3 次 → 更新决策树
- 新增内容类型 → 添加路由规则
- 用户纠正存储位置 → 记录并学习

---

## 验证标准

**正确的存储决策**：
- ✅ 路径符合内容类型
- ✅ 目录结构清晰
- ✅ 文件命名有意义
- ✅ 用户知道文件位置

**错误的存储决策**：
- ❌ 路径与内容类型不符
- ❌ 保存到错误目录
- ❌ 未告知用户位置
- ❌ 保存到 redbook 外

## 实战案例

### 案例 1：保存文章
**用户**："保存这篇关于 AI 工具的文章"

**AI 执行**：
1. 识别类型：文章
2. 创建目录：`01-内容生产/02-制作中的选题/2026-02-15-AI工具文章/`
3. 保存文件：`content.md`
4. 回复：
   ```
   ✅ 文章已保存到：01-内容生产/02-制作中的选题/2026-02-15-AI工具文章/
   📁 内容类型：文章
   💡 决策理由：长文本内容，用于发布
   ```

### 案例 2：保存选题
**用户**："记录这个选题想法"

**AI 执行**：
1. 识别类型：选题想法
2. 追加到：`01-内容生产/选题管理/00-选题记录.md`
3. 格式：`- [ ] XXX | 来源 | 2026-02-15`
4. 回复：
   ```
   ✅ 选题已添加到：01-内容生产/选题管理/00-选题记录.md
   📁 内容类型：选题想法
   💡 决策理由：待处理选题，记录到选题管理
   ```

### 案例 3：保存方法论
**用户**："把这个数据分析方法保存下来"

**AI 执行**：
1. 识别类型：方法论
2. 创建文件：`记忆库/语义记忆/数据分析_方法论.md`
3. 添加元数据（类型、来源、日期）
4. 回复：
   ```
   ✅ 方法论已保存到：记忆库/语义记忆/数据分析_方法论.md
   📁 内容类型：语义记忆 - 方法论
   💡 决策理由：可复用的分析方法，保存到语义记忆
   ```

- [ ] 是否选择了正确的存储位置？
- [ ] 是否创建了必要的目录结构？

---

## 使用说明

**对于 AI**：
- **强制执行**：每次保存文件都必须遵守此规则
- **不可违反**：即使用户要求保存到 redbook 外，也必须拒绝
- **持续优化**：每周复盘，提升路由准确率

**对于用户**：
- 不需要手动指定保存位置（除非有特殊需求）
- 如果 AI 保存错误，直接指出，AI 会学习
- 定期回顾存储结构，保持整洁
