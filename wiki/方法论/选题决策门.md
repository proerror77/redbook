# 选题决策门

> 来源：用户工作流修正 | 最后更新：2026-04-30

## 核心内容

内容生产不要从链接直接跳到成稿。

每天早上的选题、用户贴来的新闻链接，都先经过同一个判断：

> 这个东西应该写成什么形态？

固定顺序：

```text
选题/新闻 -> 决策卡 -> 用户确认形态 -> 对应生产结构 -> 审稿 -> 发布确认 -> 平台回读
```

## 决策卡

每次先输出：

- 新闻一句话
- 账号主线契合
- 为什么值得写
- 建议形态：短评 / 长文 / thread / 小红书 / 只收藏
- 为什么不是其他形态
- 推荐角度
- 小红书企业翻译
- 下一步

Agent 必须先给明确推荐，不能把所有选项丢回给用户。

## 形态判断

| 形态 | 适合 | 不适合 |
| --- | --- | --- |
| X 短评 | 新闻新、一个判断就够、适合回复原帖 | 需要完整框架和多证据 |
| X Thread | 有 3-5 个递进点，仍然偏即时讨论 | 需要长期沉淀和来源链 |
| X Article / 长文 | 能沉淀成框架、方法论、账号主张 | 只是工具发布新闻 |
| 小红书图文 | 能翻译成企业/商业 AI 应用、管理者清单、ROI/风险 | 只是开发者工具细节或 crypto 噪音 |
| 只收藏 | 有价值但不够贴合账号，或没有独特判断 | 用户明确要发布 |

## 与跨平台账号编排的关系

`选题决策门` 解决“写不写、写成什么”。

`跨平台账号编排` 解决“同一个命题在不同平台怎么翻译”。

两者顺序不能反：

1. 先决定形态。
2. 再决定平台翻译。
3. 再生产稿件和图文。
