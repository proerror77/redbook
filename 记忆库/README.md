# 三层记忆架构

这是基于认知科学的三层记忆系统，让 AI 助手能够记住、学习和进化。

## 架构说明

```
记忆库/
├── 情景记忆/          # 第一层：原始经验
│   ├── 日记/          # 每日记录
│   └── 2026-02/       # 月度归档
├── 语义记忆/          # 第二层：提炼知识
│   └── （知识文档）
└── 强制规则/          # 第三层：必须遵守的规则
    └── （规则文档）
```

## 三层记忆的运作机制

### 第一层：情景记忆（Episodic Memory）
**作用**：记录具体发生的事情

**存储内容**：
- 每日对话记录
- 项目进展日志
- 问题解决过程
- 临时笔记和想法

**特点**：
- 原始、未加工
- 时间线清晰
- 可检索、可回顾

**使用场景**：
- "上周我们讨论过什么？"
- "上次这个问题怎么解决的？"
- "这个项目的进展如何？"

---

### 第二层：语义记忆（Semantic Memory）
**作用**：将经验提炼为可复用的知识

**存储内容**：
- 总结的规律和模式
- 提炼的方法论
- 领域知识库
- 最佳实践

**提炼触发条件**：
1. 发现重复模式（≥3 次）
2. 完成重大项目后
3. 主动复盘时

**提炼流程**：
```
情景记忆 → 识别模式 → 语义记忆
```

**使用场景**：
- "如何写出好的小红书标题？"（查看 `标题方法论.md`）
- "内容发布的标准流程是什么？"（查看 `发布流程.md`）

---

### 第三层：强制规则（Procedural Rules）
**作用**：固化为「必须遵守」的行为准则

**存储内容**：
- 严格执行的规则
- 不可违反的边界
- 自动化的决策逻辑

**提炼触发条件**：
1. 从语义记忆中发现「必须遵守」的规则
2. 反复出错后总结的教训
3. 系统级约束

**提炼流程**：
```
语义记忆 → 发现必须遵守的规则 → 强制规则
```

**使用场景**：
- 智能存储路由（自动判断文件保存位置）
- 任务意图捕获（自动识别时间意图）
- 发布状态更新（自动同步文件状态）

## 记忆进化示例

### 示例 1：标题创作

**情景记忆**（2026-01-10）：
> 今天写了一篇关于 AI 的文章，标题是"AI 时代的内容创作"。
> 数据：阅读量 500，点赞 10。

**情景记忆**（2026-01-15）：
> 今天写了一篇关于 AI 的文章，标题是"别卷了！AI 帮你 10 分钟搞定内容创作"。
> 数据：阅读量 5000，点赞 200。

**情景记忆**（2026-01-20）：
> 今天写了一篇关于效率的文章，标题是"效率提升的 5 个方法"。
> 数据：阅读量 800，点赞 15。

**情景记忆**（2026-01-25）：
> 今天写了一篇关于效率的文章，标题是"90% 的人不知道！这 5 个方法让你效率翻倍"。
> 数据：阅读量 8000，点赞 350。

**语义记忆** → 提炼规律：
> **标题方法论**
>
> 高转化标题公式：
> - 使用感叹号和数字
> - 制造稀缺感（"90% 的人不知道"）
> - 强调结果（"效率翻倍"）
> - 避免平铺直叙

**强制规则** → 固化为自动决策：
> **强制规则_标题生成**
>
> 每次生成标题时，必须：
> 1. 包含数字或感叹号
> 2. 使用痛点/结果导向语言
> 3. 避免抽象概念
> 4. 生成 3 个选项供选择

---

### 示例 2：文件存储

**情景记忆**（2026-02-01）：
> 用户说"保存这个文章"，我询问保存位置。
> 用户回复："01-内容生产/02-制作中的选题/"

**情景记忆**（2026-02-03）：
> 用户说"保存这个文章"，我询问保存位置。
> 用户回复："你应该知道文章要放哪里啊"

**情景记忆**（2026-02-05）：
> 用户说"保存这个视频脚本"，我询问保存位置。
> 用户不耐烦："还要我说吗？"

**语义记忆** → 提炼规律：
> **存储路由规则**
>
> 内容类型 → 存储位置映射：
> - 文章文稿 → `01-内容生产/02-制作中的选题/`
> - 选题想法 → `01-内容生产/选题管理/`
> - 素材资源 → `02-内容素材库/`
> - 已发布内容 → `01-内容生产/03-已发布的选题/`

**强制规则** → 固化为自动决策：
> **强制规则_智能存储路由**
>
> 当用户说"保存XXX"时：
> 1. 自动分析内容类型
> 2. 根据映射表确定存储位置
> 3. 创建目录和文件
> 4. 告知用户保存结果
>
> 不需要询问用户"保存到哪里"。

## 使用建议

### 对于用户
1. **不要手动整理记忆**：让 AI 自动提炼
2. **遇到重复问题时**：提示 AI "这个问题好像出现过多次了"
3. **定期复盘**：每月让 AI 回顾情景记忆，提炼新的知识

### 对于 AI
1. **主动识别模式**：当发现重复≥3次时，主动提炼
2. **渐进式提炼**：情景 → 语义 → 强制，不要跳级
3. **标注来源**：每条语义记忆/强制规则都要标注来源情景

## 文件命名规范

- 日记：`YYYY-MM-DD.md`
- 月记：`YYYY-MM-月记.md`
- 对话历史：`对话历史-YYYY-MM.md`

### 语义记忆
- 方法论：`{领域}_方法论.md`
- 知识库：`{主题}_知识库.md`
- 最佳实践：`{场景}_最佳实践.md`

### 强制规则
- 规则：`强制规则_{功能}.md`

- 整合重复的语义记忆
- 验证强制规则的有效性
