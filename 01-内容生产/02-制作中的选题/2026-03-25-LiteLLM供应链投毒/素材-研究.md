# LiteLLM 供应链投毒素材

## 核心结论

- 不要把这篇写成单纯新闻复述
- 重点应该是：`AI 团队平时最忽略的，不是模型，是供应链`
- 适合技术提醒 + 工程动作清单

## 已确认事实

### Issue #24512

- 时间：2026-03-24
- 主题：`litellm 1.82.8` 中存在恶意 `.pth` 文件
- 核心风险：
  - Python 启动即执行
  - 不需要 `import litellm`
  - 可收集大量敏感文件和凭证
  - 向 `https://models.litellm.cloud/` 外发

来源：

- https://github.com/BerriAI/litellm/issues/24512

### Issue #24518

- 维护团队时间线更新：
  - 受影响包：`1.82.7`、`1.82.8`
  - 包已删除
  - 所有 maintainer 账户已轮换
  - 新版本暂停发布，直到完成链路扫描
  - 已与 Mandiant 合作处理

来源：

- https://github.com/BerriAI/litellm/issues/24518

## 最值得写的角度

### 角度 1

AI infra 包的位置太核心，一旦出事不是功能 bug，而是系统级风险。

### 角度 2

“包管理器里有版本”不等于“这个版本可信”，必须回看 GitHub release 和构建来源。

### 角度 3

AI 团队现在本地环境里堆满各种 key，所以“只是开发机中毒”这句话已经不成立了。

## 应该给读者的动作

1. 查是否安装过 `1.82.7` 或 `1.82.8`
2. 查 `site-packages` 里是否有 `litellm_init.pth`
3. 轮换所有可能暴露的凭证
4. 回查 CI / Docker / 构建机
5. 强化依赖 pin、镜像隔离、构建验证

## 最适合打出去的一句话

**AI 时代最脆弱的地方，往往不是模型本身，而是围着模型转的那圈供应链。**
