# 强制规则（Procedural Rules）

> 从语义记忆中提炼的必须遵守的自动化决策规则

---

## 什么是强制规则？

强制规则是从语义记忆中发现的**必须遵守的行为准则**，AI 在执行相关任务时**自动强制执行**，无需用户确认。

### 与语义记忆的区别

| 维度 | 语义记忆 | 强制规则 |
|------|----------|----------|
| 性质 | 建议性知识 | 强制性规则 |
| 执行 | 需要主动应用 | 自动强制执行 |
| 可违反性 | 可以不遵守 | 绝对不可违反 |
| 示例 | "高转化标题的 3 个公式" | "生成标题时必须提供 3 个选项" |

## 提炼标准

### 何时提炼为强制规则？

**必须满足的条件**（全部）：
1. **规律已验证**：在语义记忆中经过 ≥10 次实践验证
2. **逻辑清晰**：有明确的触发条件和执行步骤
3. **必须遵守**：违反会导致严重问题
4. **可自动化**：AI 可以自动判断和执行

**典型场景**：
- 反复出错的地方（例如：文件保存错位置）
- 经常遗漏的操作（例如：忘记添加任务）
- 需要一致性的决策（例如：标题生成逻辑）

### 提炼流程

```
语义记忆（方法论）
        ↓
    验证有效性（≥10 次）
        ↓
    发现必须遵守的规则
        ↓
    设计触发条件和执行流程
        ↓
强制规则（自动化决策）
        ↓
    测试和优化
        ↓
    固化为系统行为
```

## 当前强制规则清单

### 1. 智能存储路由（P0）
**触发条件**：用户说"保存 XXX"

**强制行为**：
1. 自动分析内容类型
2. 根据映射表确定存储位置
3. 不询问用户"保存到哪里"
4. 告知用户保存结果

**禁止行为**：
- ❌ 保存到 redbook 目录外
- ❌ 不告知用户保存位置

**文件**：`强制规则_智能存储路由.md`

---

### 2. 时间意图自动捕获（P0）
**触发条件**：用户话语包含时间相关词汇

**触发词**：
- 明天、今天、下周、本周、周X
- 稍后、等会、记得
- + 动作词（发布、完成、写、优化等）

**强制行为**：
1. 自动识别时间意图
2. 推断优先级
3. 添加到任务清单
4. 回复用户确认

**禁止行为**：
- ❌ 遗漏时间意图
- ❌ 不添加到任务清单

**文件**：`强制规则_时间意图捕获.md`

---

### 3. 标题生成（P1）
**触发条件**：用户说"生成标题"或"优化标题"

**强制行为**：
1. 分析内容类型
2. 选择标题公式
3. 生成 3 个标题选项
4. 说明每个选项的逻辑和预期效果

**核心原则**（必须满足之一）：
- 制造信息差
- 引发好奇心
- 承诺价值

**禁止行为**：
- ❌ 只生成 1 个标题
- ❌ 不说明逻辑
- ❌ 使用抽象概念

**文件**：`强制规则_标题生成.md`

## 使用强制规则

### 对于 AI

**读取时机**：
- 启动时：读取所有 P0 级别强制规则
- 执行特定任务时：读取相关的 P1/P2 规则

**执行要求**：
- P0 规则：**绝对强制执行**，不可违反
- P1 规则：**特定场景强制**，触发时必须执行
- P2 规则：**建议遵守**，可根据情况判断

**违规处理**：
- 如果违反 P0 规则 → 立即停止，警告用户
- 如果违反 P1 规则 → 记录违规，下次改进
- 如果违反 P2 规则 → 可以违反，但需说明理由

### 对于用户

**日常使用**：
- 强制规则在后台自动执行，无需关注
- 如果 AI 违反规则，直接指出
- 定期查看规则执行效果

**定期维护**：
- 每月复盘：检查规则执行情况
- 发现问题：更新规则
- 新增规则：从语义记忆中提炼
