# 一个月涨 1 万粉，我没洗过一篇稿：Claude Code + Obsidian 原创系统公开

![](https://pbs.twimg.com/media/HG7FkvCbQAAgS35.jpg)

一个月从零涨到 1 万粉丝。

我没有洗过稿。

不是因为我反感或讨厌洗稿，而是因为洗稿虽然能帮我快速摸通脉络，但没办法帮我长期成长。

所以我愿意在最开始的时候难一点。但当我真正实践之后，我发现——它可能并不难。

今天把这套方法完整公开：让你一边成长，一边原创，一边搞流量。而这三者不是孤立的，因为你成长了，所以你可以原创，所以自然而然有流量。而流量会倒逼着你去成长，倒逼着你去原创。

## 省流版工作流

输入层：录音语料 + 阅读思考 + 日常想法
↓
 处理层：Obsidian本地存储 + Claude Code本地调用
↓
 输出层：三个指令（challenge/emerge/draft）挖掘金句和真知灼见
↓
 价值闭环：成长 → 原创 → 流量 → 倒逼成长

核心工具：Claude Code + Obsidian + 自定义Command

下面逐步拆解。

## 先看效果

我每天只需要在Claude Code里输入一个指令，它就能对我过去的全部信息：

1. 提出挑战：我在回答挑战的过程，就是遇到新问题，迸发出新答案的过程
2. 挖掘隐藏想法：挖掘出我内心可能想说，但没有说出口的话，指引我接下来的方向
3. 监督执行：基于时间顺序对比前后执行情况，告诉我是执行得好还是坏

![](https://pbs.twimg.com/media/HG6_6uQbwAAriYN.png)

在这个过程中，我一边成长，一边和AI碰撞出一系列金句和真知灼见。

这些真知灼见就是我的内容素材源泉——我用自己的话重复一遍，或者发表自己的看法，发到推特上获取流量。

## 为什么大部分人做不到？

我见过最印象深刻的一个学员：

我教完他AI知识库的知识之后，下一次问他："你回去学到了什么？"

他说："我创建了100个知识库。"

我问："这100个知识库里放了什么东西吗？"

他说："现在100个知识库都是空的，我以后一定会放的。"

但我知道他以后大概率不会放。

这就是问题所在：

1. 为了学而学：搭建了一堆系统，但没有真材实料的内容
2. 为了抓而抓：疯狂收藏文章，但从不产生自己的想法
3. 没有输出习惯：不记录、不思考、不沉淀

我的解决方案是：从真实输入开始。

## 第一步：建立你的语料库（耗时：每天10-30分钟）

如果你觉得自己"没有想法"，我推荐几个路径：

1. 录音语料

用千问 app 把自己一天全部的沟通录下来——和别人的对话、听到别人说的话，全部作为你的语料库。

2. 阅读

你在看别人文章的过程中，也会产生一系列想法。看完一个内容之后，你但凡有这个意识，不可能一点想法也没有。

3. 日常所思所想

真正的一些思考和想法，直接记录在 Obsidian 即可。

这些东西都是你原创的内容，都将作为你未来的语料库。

## 第二步：搭建本地知识系统（耗时：0.5-1小时）

为什么选择Obsidian + Claude Code？

Obsidian的优势：

- 完全本地的笔记软件，不需要担心隐私问题
- 所有文件都是以Markdown格式保存在本地

Claude Code的优势：

- 不再是浏览器中的聊天助手，而是根植在你的电脑
- 能够对你本地文件进行编辑的软件

两者结合，产生神奇的效应。

相比之下，春节期间爆火的 OpenClaw，除主动执行任务外，在 Claude Code + Obsidian 面前基本处于拉胯状态。龙虾的长期记忆在 Claude Code + Obsidian 的联动下，完全被吊打。

安装步骤

1. 下载Obsidian

- ✈️点我跳转 Obsidian 官网下载，按照指引创建知识库
- 核心价值就是记笔记，不需要关注插件、模板、主题
- 那些都是外表，都是形式，解决不了本质问题

2. 安装Claude Code

- 使用 Trae 的方式安装（参考我之前的文章：靠上门安装 OpenClaw，这个月我赚了六位数。）
- 遇到任何问题，复制粘贴或截图发送给Trae，让Trae帮你修改

3. 切换API

- 很多人订阅不了官方会员，推荐使用 CodeSome 的 API

![](https://pbs.twimg.com/media/HG6_0gvagAAcQk0.png)

- 是市面上公认性价比最高的，我见过真人，至少不会跑路
- 使用 CC Switch 软件（✈️ 点我跳转 ccswitch 官网）填写相应信息，丝滑切换 API
- 不知道怎么填可以问客服，或者截图发给 AI 让 AI 告诉你，这个过程是锻炼我们 AI First 的过程。

## 第三步：精准定位到知识库（耗时：1分钟）

你需要多多少少了解一些命令行知识。

默认情况下打开终端，你是在电脑的根目录打开的。我们需要精准定位到Obsidian知识库。

最简单的方式：

1. 同时打开访达和终端
2. 在终端输入cd （cd后面有一个空格，如果只能学一个终端指令，那么它就是 cd。）
3. 把Obsidian知识库文件夹（也可以是其子文件夹）拖进终端，
4. 按回车
5. 输入 claude
6. 一路同意

![](https://pbs.twimg.com/media/HG6_OOoaQAAGHYy.png)

这样你就跳到了Obsidian里，并且能在当前文件夹够调用Claude Code了。

你的路径和我的路径不一样没关系，因为根目录名称和文件夹名称都不同，只要是拖过来的就行。

## 第四步：安装自定义指令（耗时：2分钟）

我把我的指令推送到了GitHub网站上，大家可以直接复制粘贴代码发送给Claude Code，它就会帮你安装相应的指令。

安装好之后，你只需要输入一个斜杠 /，然后输入指令名称，它就能自动运行。
过程中他会申请执行任务，直接按回车通过即可。

 三个核心指令：

1. /challenge（挑战指令）：AI会基于你的知识库给你提出挑战性问题
2. /emerge（涌现指令）：挖掘出你一直在想但从未明确说出来的想法
3. /draft（监督执行）：基于时间顺序对比前后执行情况，告诉我是执行得好还是坏

## 第五步：开始对话，产出内容（耗时：每天20-40分钟）

每次对话过程中，AI在最开始挖掘出来的点是大而泛的。

你可以针对某一个具体的点，不断和它深聊，不断表明自己具体的背景信息。

在这个过程中：

- 一方面，你在成长
- 另一方面，你和它在碰撞的过程中会产生出一系列金句、一系列真知灼见

这些真知灼见都将作为你的内容素材源泉。

你完全可以把它用你自己的话再重复一遍，或者发表你自己的看法，发到推特上去获取流量。

还有一个技巧：你可以直接跟它说，"把当前的全部对话统一以 Markdown 的形式保存下来"，它就会保存下来。在下一次调用的时候，把上一次的对话作为你自己的知识库。

## 这套系统的长期价值

1. 不再是凭空产出

当前的这个AI相较于传统浏览器窗口里的AI：

- 不再是单纯给我凭空产出结果
- 不再需要我在最开始给它补足充足的背景信息
- 所有的文档在未来都将作为我充足的背景信息

2. 每一条内容都是原创

每一条内容都是我自己基于我自己背景信息原创出来的。

我不需要担心它是否和别人重复，也不需要担心我是不是会被判定为洗稿。

3. 在输出中成长

无论是AI帮我发出挑战、表明我自己也没有说出来的话，还是监督我的执行情况，它本身是一种监督，是一种反思和复盘。

我们在这个反思和复盘的过程中就能够获得成长。

4. 公开承诺倒逼执行

当我们把所有的真知灼见逐渐发布到网上，以公开的形式公布给大家：

有的时候你公布的是你现在还没有成为的样子，是你梦想成为的那个样子。

但一旦你公开出来，它就将作为你的一份承诺，也作为大家对我们的反向监督。

它会倒逼着我们成长。

5. 形成正向循环

从此之后我们就能够实现：成长 → 原创 → 流量变现 → 商业模式，形成一个有效的正向循环。

## 最后一个关键点：倒逼输入

你会发现，当你使用不同的 AI，不断输入指令的时候，但原有的素材有限的情况下，你最终获得的挑战和信息大概率是近似相同的。

当相同的那一瞬间产生了之后，它会倒逼着你在最开始进行输入。

而当你开始进行输入的那一刻起，你就正式步入到了成长蜕变的阶段。
