# Writing Style Skill 设计方案

## 1. 目标

创建一个持久化的个人写作风格 Skill，解决 AI 生成内容的同质化问题。

## 2. Skill 结构

### 文件组织

```
~/.claude/skills/writing-style/
├── SKILL.md                    # Skill 主文件
├── references/
│   ├── style-definition.md     # 风格定义（4 部分结构）
│   ├── sample-articles/        # 用户原创文章样本

│   │   ├── sample-2.md
│   │   └── sample-3.md
│   └── iteration-log.md        # 迭代记录
```

## 2. 风格要点 (Style Points)

### 核心原则（3-5 条）

每条包含：原则 + 示例对比

#### 原则 1: 用数据说话，而非形容词堆砌

❌ **AI 味写法**:
> AI 效率非常高，能极大提升生产力，让工作变得更加高效。

✅ **我的风格**:
> 用 AI 后，一天写 200 条 listing，以前团队三个人干一周。

**要点**：
- 用具体数字代替形容词
- 前后对比 > 单纯夸赞
- 案例 > 观点

---

#### 原则 2: 短句 + 节奏感

❌ **AI 味写法**:
> 在使用 AI 工具的过程中，我们发现虽然它能够提高效率，但是如果没有合理的架构设计，最终产出的内容质量并不理想。

✅ **我的风格**:
> AI 效率是真的强。但大多数人用不好。问题不在 prompt，在架构。

**要点**：
- 句子长度 < 20 字
- 断句创造节奏
- 一句话一个观点

---

#### 原则 3: 口语化表达，避免书面语套话

❌ **AI 味写法**:
> 综上所述，我们可以得出结论...
> 随着技术的不断发展...
> 在此背景下...

✅ **我的风格**:
> 说白了就是...
> 我有个朋友...
> 这事儿很简单...

**要点**：
- 用"你/我"，不用"我们"
- 说人话，像聊天
- 避免"综上所述""随着""在此背景下"等套话

---

#### 原则 4: 结构清晰，逻辑链短

❌ **AI 味写法**:
> 首先...其次...再次...最后...综上所述...

✅ **我的风格**:
> 问题是什么？→ 为什么会这样？→ 怎么解决？→ 举个例子。

**要点**：
- 用问题驱动，而非并列结构
- 每个部分独立完整
- 逻辑链 ≤ 3 层

---

#### 原则 5: 有态度，不中立

❌ **AI 味写法**:
> AI 工具有其优势，也存在一定的局限性，需要根据具体情况灵活使用。

✅ **我的风格**:
> 不会架构的人，用 AI 就是高效生产一堆散装零件。

**要点**：
- 明确立场，不和稀泥
- 用比喻增强观点
- 允许极端表达（但有数据支撑）

---

## 3. 禁止清单 (Forbidden List)

### AI 味词汇 → 替换方案

| AI 味词汇 | 替换为 |
|-----------|--------|
| 综上所述 | 说白了 / 总结一下 |
| 随着...的发展 | 现在 / 去年 / 最近 |
| 在此背景下 | 所以 / 因此 |
| 不断地 | （删除，用具体数据） |
| 显著提升 | 翻了 X 倍 / 从 A 到 B |
| 极大地 | （删除，用具体案例） |
| 深入探讨 | 讲清楚 / 说明白 |
| 值得注意的是 | 注意 / 重点是 |
| 总而言之 | 总结 / 结论 |
| 我们可以看到 | 你会发现 / 数据显示 |

### 套话黑名单

❌ **禁止使用**：
- 在当今这个快速发展的时代...
- 随着科技的不断进步...
- 毫无疑问...
- 众所周知...
- 不言而喻...
- 首先...其次...再次...最后...
- 一方面...另一方面...
- 由此可见...
- 综合来看...

### 标点符号规则

✅ **允许**：
- 短句 + 句号（节奏感）
- 问号（引导思考）
- 省略号...（制造悬念）
- 破折号 —（补充说明）
- 顿号、（并列）

❌ **避免**：
- 分号；（太书面）
- 冒号：后面跟长段落（拆成短句）

## 4. 参考资料 (Reference Materials)

### 术语库

**专业术语使用规则**：
- AI → 直接用，不解释
- LLM → 首次使用注释"大语言模型"
- prompt → 保留英文，必要时注"提示词"
- MVP → 首次使用注释"最小可行产品"

### 常见错误模式

#### 错误 1: 开头太慢，铺垫过长

❌ **错误示例**:
> 在当今这个 AI 飞速发展的时代，越来越多的企业开始探索 AI 在业务中的应用。然而，我们发现...（前 50 字都是废话）

✅ **正确示例**:
> AI 效率是真的强。但大多数人用不好。（直接切入核心）

---

#### 错误 2: 结尾太虚，没有行动指引

❌ **错误示例**:
> 总之，AI 是一个值得探索的方向，希望大家都能找到适合自己的方法。

✅ **正确示例**:
> 三个练习方法：1. 拆解...  2. 排序...  3. 抽象...  下次任务直接用。

---

#### 错误 3: 缺少具体案例

❌ **错误示例**:
> 架构能力很重要，能显著提升效率。

✅ **正确示例**:
> 我有个朋友做跨境电商，去年开始用 AI 写产品描述。一开始很兴奋：一天生成 200 条 listing，以前团队三个人干一周。三个月后，转化率反而降了。原因很简单：200 条 listing 长得一模一样。

---

### 语气示例库

#### 场景 1: 开篇钩子

✅ **推荐模板**：
- "X 是真的 Y。但大多数人 Z。"
- "我有个朋友 X，结果 Y。原因很简单：Z。"
- "X 的人月入 Y，Z 的人还在 A。问题在哪？"

#### 场景 2: 观点表达

✅ **推荐模板**：
- "说到底就是 X。"
- "这事儿的本质是 X，不是 Y。"
- "X 替代 Y，放大 Z。"

#### 场景 3: 行动建议

✅ **推荐模板**：
- "X 怎么练？1. ... 2. ... 3. ..."
- "与其焦虑 X，不如想想 Y。"
- "下次 X 的时候，先 Y，再 Z。"

## 更新机制

### 迭代流程（基于 @dotey 方法论）

1️⃣ **AI 生成初稿**（使用当前 Style Skill）
2️⃣ **你手动编辑**（改成你满意的版本）
3️⃣ **Git diff 对比**（或 AI 对比两个版本）
4️⃣ **提取修改模式**（AI 分析你改了什么）
5️⃣ **更新 Style Skill**（将模式加入风格定义）

### 迭代记录模板

```markdown
## 迭代 #1 - 2026-02-15

**原始 AI 生成**：
> [AI 初稿片段]

**手动修改后**：
> [你修改后的版本]

**修改模式识别**：
- 删除了"随着...的发展"开头
- 将长句拆成 3 个短句
- 添加了具体数据案例

**更新到 Style Skill**：
- ✅ 已添加到"禁止清单"：避免"随着...的发展"
- ✅ 已添加到"风格要点 2"：短句示例
```

```

↓
调用 Writing Style Skill
↓
生成符合你风格的推文
```

### 场景 2: 文章润色

```
/writing-style polish [文章文件]
↓
基于你的风格重写
↓
返回修改建议 + 对比
```

### 场景 3: 翻译保持风格

```
/writing-style translate [英文文章]
↓
翻译 + 风格适配
↓
输出中文版（保持你的语气）
```

## 初始化步骤

### 步骤 1: 提供原创样本

从你的已发布内容中选择 3-5 篇最满意的文章：
- 至少 1 篇长文（1000+ 字）
- 至少 2 篇短文/推文
- 覆盖不同主题

### 步骤 2: AI 分析风格

使用 `skill-creator` agent 分析：
```
请分析这些文章的写作风格，生成 Writing Style Skill
```

### 步骤 3: 手动调优

检查生成的 4 部分内容，补充：
- 你的独特表达习惯
- 你常用的比喻
- 你的禁忌词汇

### 步骤 4: 测试迭代

用 Skill 生成 1-2 篇内容，手动修改，提取模式，更新 Skill。

## 成功标准

经过 10 次迭代后：
- ✅ AI 生成的内容，90% 以上无需大改
- ✅ 读者分不清是你写的还是 AI 写的
- ✅ AI 比你自己更一致地遵循你的风格

---

## 注意事项

1. **不要追求完美主义**：第 1 版 Skill 肯定不完美，重点是快速迭代
2. **风格会进化**：每 3 个月回顾一次，你的风格可能已经变了
3. **保持真实性**：AI 应该放大你的风格，而非创造一个虚假人设
4. **组合使用**：Writing Style 是"底味"，其他 Skills 是"菜式"
