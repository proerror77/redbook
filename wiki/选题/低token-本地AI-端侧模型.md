# 低 token、本地 AI、端侧模型

> 最后更新：2026-04-06
> 来源：`05-选题研究/2026-04-06-低token-本地AI-端侧模型-深度研究.md`

## 核心判断

这不是三个零散话题，而是一条连续演进路线：

- `低 token`：先把同一个模型的废话砍掉，降低输出成本和延迟
- `本地 AI`：再把适合的任务从云 API 拉回自己的机器
- `端侧模型`：最后把最靠近用户、最重隐私、最讲实时性的任务推到设备端

一句话：

**AI 正从“比谁更强”转向“比谁更省、谁更快、谁离用户更近”。**

## 为什么值得长期跟

### 信号 1：token 开始变成经营指标

- `Caveman` 这种项目能火，说明重度用户已经不再满足于“回答得像人”
- 他们开始要求：更短、更快、更便宜
- 对 agent 工作流来说，output token 不只是表达风格，而是速度和成本

### 信号 2：本地模型长出了工程接口

- Ollama 已经正式支持 `Claude Code`
- LM Studio 已经提供 `OpenAI-compatible` / `Anthropic-compatible` endpoint
- 本地模型开始能被 agent、脚本、CI、MCP 调用

这说明本地 AI 正从“发烧友玩具”变成“可接入基础设施”。

### 信号 3：端侧 agent 开始出现产品样板

- Google 已经把 `Gemma 4 + AI Edge Gallery + LiteRT-LM` 连成一条线
- 官方口径直接讲 `agentic skills`
- App Store 版本已经上线 `Gemma 4`、`Agent Skills`、`Thinking Mode`

这说明端侧不是只做 demo，而是在试探新一层产品形态。

## 最强内容观点

- AI 下一阶段不再只是拼模型 IQ，而是拼部署位置
- token 正在变成 AI 产品的新毛利率
- 本地 AI 的价值不是“离线聊天”，而是把高频便宜活拿回来
- 端侧模型的终局不是小号 ChatGPT，而是设备上的 agent runtime
