# X 每日选题分流与知识沉淀

> 来源：用户定位修正 | 最后更新：2026-04-28

## 核心定位

`@0xcybersmile` 的主线不是泛 AI 新闻号，也不是纯 Crypto 快讯号。

主线应稳定围绕：

- AI Agent
- AI 如何导入企业
- AI 协作方式变化
- workflow / coding agent / browser agent / email agent / voice agent
- 独立开发者与小团队如何用 AI 提效
- 可执行的小技巧、检查清单、工具使用经验

AI 与 Crypto 时事可以写，但它们应该服务于账号主线：帮助读者理解 AI 工具、agent 工作流、企业导入、自动化协作、风险边界和新基础设施，而不是变成泛资讯搬运。

## 每日选题分流

每天从 X.com、HN、Reddit、官方博客、项目发布、Crypto 新闻源中抓到的素材，先分成四条 lane。

### Lane A：主线内容

占比建议：50% - 60%。

适合写：

- AI Agent 进入真实工作流
- 企业如何导入 AI，而不是只买工具
- 员工、团队、管理者如何和 agent 协作
- coding agent、browser agent、email agent、voice agent 的产品形态
- AI 工具 ROI、token 成本、验证成本、权限边界
- 小团队 / 独立开发者的 agent 工作流技巧

判断标准：

1. 是否和 `agent / workflow / 企业导入 / 协作方式` 直接相关？
2. 是否能给读者一个可执行动作？
3. 是否能沉淀进 Wiki，变成长期资产？

### Lane B：AI 时事评论

占比建议：25% - 30%。

适合写：

- OpenAI / Anthropic / Google / xAI / Meta 的新模型、新工具、新策略
- Codex、Claude Code、Cursor、OpenCode、Hermes 等工具更新
- AI infra、沙箱、权限、审计、浏览器、代码图谱、上下文管理
- 大厂裁员、AI native 组织、AI builder 岗位变化

写法要求：

- 不只复述新闻
- 必须回答“这对 agent / 企业导入 / 协作方式意味着什么”
- 尽量给一个产品判断或使用建议

### Lane C：Crypto / AI x Crypto 时事

占比建议：10% - 15%。

适合写：

- AI x Crypto 基础设施
- 预测市场、Polymarket、oracle、结算机制
- agent 交易、自动化策略、钱包权限、链上风控
- Crypto 项目如何使用 AI 做运营、交易、数据分析、风控
- 与独立开发者、AI 工具、自动化工作流有关的 Crypto 新闻

写法要求：

- 只写和账号主线有关的 Crypto
- 泛行情、价格涨跌、meme 热点默认跳过
- 如果写 Crypto 新闻，必须有可信来源链接
- 最好把它转成一个更大的问题：oracle、权限、自动化、结算、风险、信息差

### Lane D：轻量互动 / 生活化观察

占比建议：5% - 10%。

适合写：

- 使用 AI 工具时的小体感
- token 烧得快、本地机器发热、agent 跑偏、验证成本变高
- 对 X timeline 里某个现象的短观察
- 轻量提问，测试读者反馈

这条 lane 的目标不是建立完整观点，而是提升账号的人味和互动密度。

## 每日选题评分

每个候选题按 5 个维度打分，满分 10 分。

| 维度 | 问题 | 权重 |
|------|------|------|
| 主线相关性 | 是否贴合 AI Agent / 企业导入 / 协作方式？ | 30% |
| 时效性 | 是否适合今天写，而不是下周也一样？ | 20% |
| 互动性 | 是否容易引发回复、转发、争议或收藏？ | 20% |
| 证据强度 | 是否有来源、截图、实测、数据或高互动样本？ | 15% |
| Wiki 价值 | 是否能沉淀成长期知识页、框架或案例？ | 15% |

推荐阈值：

- 8 分以上：当天优先写
- 6-8 分：进入待深化
- 6 分以下：只存素材，不写

## 每日输出结构

每天早报或人工研究输出时，最终应给出：

1. 主线候选 2-3 个
2. AI 时事速评候选 1-2 个
3. Crypto / AI x Crypto 候选 0-1 个
4. 今日最推荐写的 1 个
5. 需要进入 Wiki 的知识点 3-5 条

示例：

```text
今日最值得写：
Agent 入口从聊天框进入浏览器 / 邮箱 / 代码图谱

为什么：
- Timeline 多人讨论
- 和账号主线高度相关
- 可沉淀到 AI Agent 企业导入与协作
- 可产出检查清单：企业导入 agent 前要看哪些入口和权限
```

## LLM Wiki 沉淀流程

每日研究结束后，不只是生成早报，还要把信息进入 Wiki。

### 1. Ingest

把当天素材写入相关页面：

- AI Agent / 企业导入 / 协作方式 → `wiki/选题/AI Agent企业导入与协作.md`
- AI 工具、效率、成本、workflow → `wiki/选题/AI工具与效率.md`
- Crypto / AI x Crypto → `wiki/选题/Crypto与AI时事评论.md`
- 写法结构、hook、互动模式 → `wiki/方法论/`
- 可复用表达 → `wiki/素材/金句库.md` / `wiki/素材/框架库.md`

### 2. Query

写作前必须先 query 相关 Wiki 页面，回答：

- 这个题之前写过吗？
- 已有的核心判断是什么？
- 哪些表达可以复用？
- 今天的新信息和旧判断有什么增量？

### 3. Attach

如果进入内容生产 run，Wiki query 结果应 attach 到 run，作为 research 证据。

### 4. Lint

每天或每周检查：

- 是否出现重复页面
- 旧判断是否被新事实推翻
- AI / Crypto 时事是否被沉淀成长期框架，而不是散落在日报里

## 后续写作原则

1. 每天可以追热点，但主线不能漂。
2. AI 时事要转成 agent / workflow / 企业导入的判断。
3. Crypto 时事要转成 oracle / 权限 / 自动化 / 结算 / 风控的判断。
4. 每篇内容至少有一个可沉淀进 Wiki 的知识点。
5. 不写“新闻发生了什么”，要写“这对使用 AI 工作的人意味着什么”。
