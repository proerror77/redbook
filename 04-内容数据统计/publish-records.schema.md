# Publish Records JSONL Schema

> Canonical ledger: `04-内容数据统计/publish-records.jsonl`

`数据统计表.md` is a human-readable report view. New publish evidence should first be recorded as JSONL so T+0/T+1/T+3 updates can be appended without hand-editing markdown tables.

## Stages

- `T+0`: immediate publish evidence. Required: `platform`, `title`, `published_at`, and either `status_url` or `note_id`.
- `T+1`: next-day metrics update. Recommended: `views`, `likes`, `reposts`, `replies`, `comments`, `shares`, `saves`.
- `T+3`: retention/retrospective update. Recommended: result judgment, method-learning note, and whether to update `wiki/方法论/`.

## Fields

| Field | Type | Notes |
| --- | --- | --- |
| `record_id` | string | Stable id for one post/content item. Reuse the same id across T+0/T+1/T+3. |
| `stage` | string | One of `T+0`, `T+1`, `T+3`. |
| `platform` | string | Examples: `x.com`, `xiaohongshu`, `wechat`. |
| `title` | string | Human-readable title or short content summary. |
| `published_at` | string | ISO-like datetime with timezone when available. |
| `status_url` | string | X status URL, WeChat URL, or public content URL. |
| `post_id` / `note_id` | string | Platform stable id when available. |
| `account` | string | Publishing account or handle. |
| `content_path` | string | Repo-relative path to content artifact. |
| `publish_record_path` | string | Repo-relative path to detailed markdown publish evidence. |
| `metrics` | object | Numeric metrics such as `views`, `likes`, `reposts`, `replies`, `comments`, `shares`, `saves`, `followers`. |
| `source_urls` | string[] | Source/news/product links used by the post. |
| `evidence` | string[] | Verification evidence labels, e.g. `status_page_visible`, `homepage_top_post`, `note_management_reviewing`. |
| `notes` | string | Free-form short note. |
| `recorded_at` | string | Local append time generated by `tools/record_publish.py`. |

## CLI

Append a T+0 record:

```bash
python3 tools/record_publish.py \
  --stage T+0 \
  --platform x.com \
  --title "short title" \
  --published-at "2026-04-28T05:42:00+08:00" \
  --status-url "https://x.com/..." \
  --content-path "01-内容生产/02-制作中的选题/.../X短评.md" \
  --publish-record-path "01-内容生产/02-制作中的选题/.../发布记录.md" \
  --views 2 \
  --evidence status_page_visible
```
