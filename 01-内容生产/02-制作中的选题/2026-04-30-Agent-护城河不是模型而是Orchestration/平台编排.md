# 平台编排

> 对应选题：Agent 护城河不是模型，而是 Orchestration
> 日期：2026-04-30

## 核心命题

这个选题真正要证明的是：

> 企业 AI 的竞争，不在谁接了最强模型，而在谁能把 AI 接进真实业务流程。

它服务账号主线的方式是：

> 把 Cursor SDK / Warp / agent orchestration 的开发者信号，翻译成企业导入 AI 时必须关注的 workflow、权限、review、审计、回滚和组织记忆问题。

## 平台角色

| 平台 | 读者任务 | 写法 | 不写什么 |
| --- | --- | --- | --- |
| X.com | 看懂 agent 平台竞争层级变化 | Cursor SDK vs Warp 作为产业信号，讲模型能力 -> orchestration 能力 | 不写工具教程，不写产品清单 |
| 小红书 | 判断企业 AI Agent 是否能落地 | 用老板/管理者能懂的 7 问清单，落到业务流程和风险控制 | 不解释 Cursor / Warp 新闻细节，不拆长文段落 |
| 公众号/长文 | 保存完整论证和来源 | 讲清 agent SDK、control plane、orchestration、企业评估框架 | 不做短平快热点 |

## 小红书翻译方向

标题候选：

- 企业用 AI，别只问模型强不强
- 真正能落地的 AI Agent，都有这 7 个特征
- 老板选 AI 工具，先问这 7 个问题

7 张卡片建议：

1. 封面：企业用 AI，别只问模型强不强
2. 痛点：很多 AI 项目失败，不是模型不够强
3. 核心判断：企业买的不是回答能力，是流程执行能力
4. 7 问清单：任务、上下文、权限、审核、恢复、审计、沉淀
5. 对比：聊天机器人 vs 企业 Agent
6. 场景：销售线索、客服工单、财务对账、招聘筛选
7. 行动：先选一个流程试点，不要先买一堆工具

## 小红书审稿标准

- 每张卡只承担一个企业读者任务。
- 读者不需要知道 Cursor / Warp 的细节，也能理解判断。
- 至少出现 3 个企业关键词：业务流程、权限、审核、ROI、风险、沉淀、试点。
- 结尾必须给行动建议：选一个低风险流程，定义输入、权限、review、输出和复盘指标。
