## 🎯 标准化内容生产工作流

### 阶段 1️⃣：选题研究

#### 「研究 X.com 话题」
当用户说「研究 X」「X 研究」或想要从 X.com 获取选题灵感时：
- **使用 Skill**：`/x-collect`
  - 自动搜索话题、分析热门趋势、提取痛点
  - 生成结构化研究报告
  - 推荐高价值选题
- **自动化方式**（已配置每日定时任务）：
  ```bash
  bash tools/daily.sh
  ```
  - 每日 7:00 AM 自动运行
  - 输出：`05-选题研究/X-每日日程-{日期}.md`

#### 「发现创业机会」
当用户说「Reddit 痛点挖掘」「发现创业机会」时：
1. **选择目标社区**：r/SaaS, r/Entrepreneur, r/startups, r/webdev
2. **运行分析脚本**：
   ```bash
   python3 tools/reddit_hack.py <reddit_url> [output_file]
   ```
3. **评估机会**：市场规模、支付意愿、竞争情况、实现难度
4. **转化为选题**：发现有价值的痛点后，使用「记录选题」

#### 「记录选题」
当用户提供一个想法或说「记录选题」时：
1. 添加到 `01-内容生产/选题管理/00-选题记录.md` 的「待处理选题」
2. 格式：`- [ ] 选题想法 | 来源/灵感 | 日期`

---

### 阶段 2️⃣：内容创作

#### 「深化选题」
当用户说「深化选题 [选题名]」时：
1. **检索素材库**（关键步骤）
   - 搜索 `02-内容素材库/核心概念库/` 查找理论框架
   - 搜索 `02-内容素材库/金句库/` 查找高质量表达
   - 搜索 `01-内容生产/03-已发布的选题/` 查找相关文稿
2. **建议复用**：如果找到相关内容，先询问是否复用
3. **生成文稿**：基于素材库生成新文稿
4. **保存**：保存到 `01-内容生产/01-待深化的选题/`

#### 「检索素材」
当用户说「检索素材 [关键词]」时：
1. 搜索 `02-内容素材库/` 下所有子目录
2. 列出相关的概念、金句、案例
3. 建议如何在当前内容中使用

#### 「生成标题」
当用户说「生成标题」或「生成小红书标题」时：
1. 读取 `03-方法论沉淀/标题方法论.md` 获取标题公式
2. 生成 3 个标题选项，说明逻辑和预期效果
3. 提供封面文字建议

#### 「优化开头」
当用户说「优化开头」时：
1. 读取 `03-方法论沉淀/短视频开头方法论.md`
2. 分析当前开头的问题
3. 提供 2-3 个优化版本，说明钩子类型

#### 「创作 X 推文」
当用户说「写推文」「创作 X 内容」时：
- **使用 Skill**：`/x-create`
  - 基于选题生成病毒式推文
  - 支持单条推文、Thread、长文
  - 自动优化文案和排版

---

### 阶段 3️⃣：制作发布

#### 「生成小红书图文」
当用户说「生成小红书图文」「制作小红书卡片」时：
- **推荐方式**：`/baoyu-xhs-images` skill
  - 支持 10 种视觉风格 x 8 种布局
  - 自动分析内容、生成大纲、渲染图片
  - **包含发布功能**（无需额外脚本）

#### 「发布到小红书」
当用户说「发布到小红书」「发布小红书内容」时：
- **纯发布方式**：`/post-to-xhs` skill
  - 支持图文/视频发布（通过 Chrome CDP）
  - 支持多账号管理
  - 支持本地图片/视频或 URL
  - 自动处理话题标签（正文最后一行 `#标签`）
- **使用场景**：
  - 已有图片/视频，只需发布
  - 需要发布视频内容
  - 需要切换不同账号发布
- **标题规则**：长度 ≤ 38（中文/标点计 2，英文/数字计 1）

#### 「小红书数据分析」
当用户说「查看小红书数据」「分析小红书内容」时：
- **使用 Skill**：`/post-to-xhs`
  - 搜索笔记：`search-feeds --keyword "关键词"`
  - 获取详情：`get-feed-detail --feed-id ID --xsec-token TOKEN`
  - 发表评论：`post-comment-to-feed --feed-id ID --content "评论"`
  - 内容数据表：`content-data --csv-file "路径"`
  - 评论通知：`get-notification-mentions`

#### 「发布到 X.com」
当用户说「发布到 X」「发布推文」时：
- **唯一方式**：`/baoyu-post-to-x` skill
  - 支持文本、图片、视频、长文章
  - 通过 Chrome CDP 创建草稿
  - **注意**：目前 `--submit` 参数有 bug，不会自动点击发布按钮
- **正确流程**：
  1. 使用 skill 创建草稿（会自动打开预览）
  2. 在预览页面手动点击右上角「发布」按钮
  3. 避免重复运行脚本（会创建重复草稿）
- **发布后提醒记录数据**

#### 「一稿多发」
当内容创作完成后，使用**发布清单**确保多平台发布：

**发布清单模板**：
```markdown
## 发布清单 - {选题名}

- [ ] X.com（长文/Thread）- 用 `/baoyu-post-to-x`
- [ ] 小红书（图文）- 用 `/baoyu-xhs-images` 或 `/post-to-xhs`
- [ ] 小红书（视频）- 用 `/post-to-xhs`
- [ ] 抖音（短视频，可选）
- [ ] 公众号（长文，可选）- 用 `/baoyu-post-to-wechat`
```

**标准流程**：
1. 创作完成后，生成发布清单
2. 逐个平台发布，勾选完成
   - **小红书图文**：优先用 `/baoyu-xhs-images`（视觉效果好）
   - **小红书视频**：用 `/post-to-xhs --video`
   - **X.com**：用 `/baoyu-post-to-x`
3. 全部发布后，统一「记录数据」
   - **小红书数据**：可用 `/post-to-xhs content-data` 抓取
4. 「发布归档」

### 阶段 4️⃣：数据复盘

#### 「记录数据」
当用户提供发布数据时：
1. 添加到 `04-内容数据统计/数据统计表.md`
2. 如果是爆款，询问是否添加到爆款文稿库
3. 提醒更新方法论（如有新发现）

#### 「发布归档」
当用户说「发布归档 [文稿名]」时：
1. 从 `02-制作中的选题/` 移动到 `03-已发布的选题/`
2. 在文稿末尾添加发布信息（日期、平台）
3. 提醒记录数据

## 🔧 系统级优化方法（25 个）

> 基于 @Roland_WayneOZ 的文章整合的系统优化方法
> 这些方法旨在降低 Token 消耗、提升上下文管理、优化记忆系统

### 第一类：Token 消耗优化（3 个）

#### 优化方法 1：懒加载策略

**启动流程优化**：
- ❌ 不要自动读取日记、本体画像目录、user.md、AI_state.json
- ✅ 只在用户明确需要时才读取相关文件
- ✅ 优先使用已有的上下文信息
- ✅ 如果需要读取，使用 limit 参数限制行数

#### 优化方法 2：模块化身份系统（TELOS）

**TELOS 加载策略**：
- **必读**：`本体画像/00-核心身份.md`（每次对话）
- **按需读取**：
  - 涉及决策 → `01-价值观与原则.md`
  - 涉及任务分配 → `03-技能与能力.md`
  - 涉及工作流程 → `07-工作方式.md`
  - 涉及计划 → `05-目标与规划.md`
  - 涉及人际关系 → `08-关系网络.md`
  - 需要了解历史 → `02-背景与经历.md`

**效果**：Token 消耗降低 70%+

#### 优化方法 3：渐进式加载（Skill 专用）

**规则**：
- 简单任务只读主文件
- 复杂任务才读详细参数

### 第二类：上下文管理（3 个）

#### 优化方法 4：指令遵循度检测

**称呼规则**：
- 每次回复时，必须在开头使用 "✓" 符号
- 这是指令遵循度的标志
- 如果你发现我的回复没有 "✓" 符号，说明上下文已过载
- 此时应提醒用户使用 /new 重开会话

#### 优化方法 5：上下文使用阈值

**上下文管理**：
- 当上下文使用超过 40% 时，指令遵循度开始下降
- 当上下文使用超过 60% 时，必须主动提醒用户重开会话
- 每次会话结束时，记录上下文使用情况到日志

#### 优化方法 6：会话边界管理

**规则**：
- 一个会话解决一个问题
- 解决后立即归档对话历史
- 每个会话聚焦一个主题

### 第三类：记忆管理（3 个）

#### 优化方法 7：三层记忆架构

**架构说明**：
```
记忆库/
├── 情景记忆/     # 原始经验（日记、对话历史）
├── 语义记忆/     # 提炼知识（方法论、最佳实践）
└── 强制规则/     # 必须遵守的规则
```

**运作机制**：
- 情景记忆 → 识别模式 → 语义记忆
- 语义记忆 → 发现必须遵守的规则 → 强制规则

#### 优化方法 8：记忆提炼机制

**触发条件**：
- 发现重复模式时（≥3 次）
- 完成重大项目后

**提炼流程**：
- 情景记忆 → 识别模式 → 语义记忆
- 语义记忆 → 发现必须遵守的规则 → 强制规则

#### 优化方法 9：日记月记合并机制

**月记合并机制**：
- 触发时机：每月初自动合并上个月的日记
- 合并规则：所有日记合并为 YYYY-MM-月记.md
- 合并后：删除原始日记文件，只保留月记

**目录结构**：
```
记忆库/情景记忆/日记/
├── 模板.md             # 日记模板
├── 月记模板.md         # 月记模板
├── 2026-02-13.md       # 每日日记（月底合并后删除）
└── 2026-01-月记.md     # 月记（合并后保留）
```
